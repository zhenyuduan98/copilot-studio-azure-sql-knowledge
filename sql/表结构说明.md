# 表结构说明

数据库共两张表：房源表 `realestate_listings` 与经纪人表 `realestate_agents`，通过 `agent_id` 外键关联。建表与插入数据的完整脚本见 [`Create-Azure-SQL-Tables.sql`](Create-Azure-SQL-Tables.sql)。

---

## 1. realestate_agents（经纪人表）

记录每位房产经纪人的联系方式，共 6 条示例。

| 列名 | 类型 | 说明 |
|------|------|------|
| `agent_id` | INT | **主键**，经纪人编号 |
| `first_name` | VARCHAR(50) | 名 |
| `last_name` | VARCHAR(50) | 姓 |
| `phone` | VARCHAR(20) | 电话（虚构测试数据）|
| `email` | VARCHAR(100) | 邮箱（虚构测试数据）|

---

## 2. realestate_listings（房源表）

记录每套在售房源的详细信息，共 30 条示例，分布在 NY、DE、PA、MA、VA、MD 等州。

| 列名 | 类型 | 说明 |
|------|------|------|
| `listing_id` | INT | **主键**，房源编号（同义词：MLS 编号 / Property ID / Reference Number）|
| `agent_id` | INT | **外键** → `realestate_agents.agent_id`，负责该房源的经纪人 |
| `address` | VARCHAR(255) | 门牌地址 |
| `city` | VARCHAR(100) | 城市 |
| `state` | VARCHAR(50) | 州（两位代码，如 `NY`/`DE`；用数据词汇表映射全称）|
| `price` | DECIMAL(12,2) | 挂牌价（美元）|
| `bed` | INT | 卧室数量 |
| `bath` | DECIMAL(3,1) | 卫生间数量（允许 0.5，如 2.5）|
| `square_feet` | INT | 建筑面积（平方英尺）|
| `listed_date` | DATE | 挂牌日期 |
| `description` | TEXT | 房源文字描述 |

---

## 3. 表关系

```
realestate_agents (1) ───< (多) realestate_listings
        agent_id  ◄────── FOREIGN KEY agent_id
```

一个经纪人可负责多套房源；每套房源对应一个经纪人。智能体回答「这套房子的经纪人联系方式」时，会顺着外键从房源表跳到经纪人表取 `phone` / `email`。

---

## 4. 示例数据概览

- **经纪人**：John Smith、Emily Johnson、Michael Brown、Sarah Davis、David Miller、Olivia Wilson（均为虚构）。
- **房源**：30 套，价格区间约 26.8 万 ~ 59 万美元，卧室 2~4 间，挂牌日期集中在 2025 年 1~5 月。
- **覆盖的州**：纽约(NY)、特拉华(DE)、宾州(PA)、马萨诸塞(MA)、弗吉尼亚(VA)、马里兰(MD)。

> 这些覆盖多个州、多种户型的数据，正好用来演示同义词、数据词汇表（州全称↔代码）和按条件筛选（卧室数、价格、城市）等能力。
