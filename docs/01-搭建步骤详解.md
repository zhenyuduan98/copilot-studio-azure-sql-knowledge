# 搭建步骤详解

从零搭建「房地产房源智能体」——把 Azure SQL 接成 Copilot Studio 知识源的完整流程。语义增强（同义词/描述/词汇表）见 [`02-知识增强配置.md`](02-知识增强配置.md)，生产环境限制见 [`03-已知限制与排查.md`](03-已知限制与排查.md)。

---

## 第 1 步：创建 Azure SQL 数据库

1. 登录 **Azure 门户**，搜索并进入 **Azure SQL**，点击新建 SQL 数据库；
2. 选择**资源组**、填写**数据库名称**、选择**计算 + 存储**规格；
3. 新建一个**服务器**（取名 + 选区域）；
4. **审阅 + 创建** → 部署完成后点 **转到资源**。

---

## 第 2 步：配置 SQL 服务器网络安全

1. 在 SQL 数据库的「概述」里，点击**服务器名**进入 SQL Server 页面；
2. 进入 **网络（Networking）→ 公共访问**，设为 **选定的网络（Selected networks）**——这样你本地才能用 SSMS 连上；
3. 在 **例外（Exceptions）** 区，勾选 **「允许 Azure 服务和资源访问此服务器」**——这一步是让 **Copilot Studio 能访问数据库**的关键。

---

## 第 3 步：安装并连接 SSMS

1. 从微软官网下载安装 **SQL Server Management Studio (SSMS)**；
2. 连接 Azure SQL Server，连接参数：
   - **服务器名**：第 1 步创建的服务器；
   - **身份验证**：Microsoft Entra MFA；
   - **用户名**：创建数据库时所用账号的 UPN（用户主体名称）；
3. 首次连接会提示把你的 **IP 加入防火墙规则**，点 **确定** 即可。

---

## 第 4 步：创建表并插入示例数据

在 SSMS 里对目标数据库运行 [`../sql/Create-Azure-SQL-Tables.sql`](../sql/Create-Azure-SQL-Tables.sql)：

- 创建 `realestate_agents`（经纪人）与 `realestate_listings`（房源）两张表；
- 房源表通过 `agent_id` 外键引用经纪人表；
- 插入 6 个经纪人、30 套房源示例数据。

表结构详解见 [`../sql/表结构说明.md`](../sql/表结构说明.md)。

---

## 第 5 步：在 Azure 注册应用（创建 Service Principal）

> 用服务主体鉴权，连接可被所有用户共享，无需每人单独建连接。

1. 在 Azure 门户进入 **应用注册（App Registrations）→ 新注册**；
2. 取个名字，完成注册；
3. 在「概述」页记下 **Client ID（应用 ID）** 和 **Tenant ID（租户 ID）**；
4. 进入「证书和密码」菜单，**新建客户端密码（Client Secret）**——
   ⚠️ **创建后立刻复制保存，密码只显示这一次！**

至此你拿到三样东西，后面 Copilot Studio 鉴权要用：**Tenant ID、Client ID、Client Secret**。

---

## 第 6 步：在 SSMS 里创建只读外部用户

把第 5 步的应用，在数据库里建成一个**只读用户**，落实最小权限。

1. 在 SSMS 里展开数据库 → 右键 **Users（用户）文件夹 → 新建用户（New User）**；
2. 用户类型选 **External user（外部用户）**，**名称必须与应用注册的名称完全一致**；
3. 在 **Membership（成员身份）** 选项卡，勾选 **db_datareader**（只读权限）。

> 只给 `db_datareader`，意味着智能体只能查询、不能修改或删除数据——这是安全上的关键。

---

## 第 7 步：在 Copilot Studio 添加 Azure SQL 知识源

1. 新建一个 **Agent（智能体）**，进入 **知识（Knowledge）选项卡 → 添加知识（Add Knowledge）**；
2. 选择 **Azure SQL** 选项；
3. 填入 **服务器名** 与 **数据库名**；
4. 鉴权类型选 **服务主体（Service principal / Microsoft Entra ID application）**，填入：
   - **Tenant ID（租户）**
   - **Client ID**
   - **Client Secret**
5. 勾选 **Listings（房源）** 与 **Agents（经纪人）** 两张表 → **添加到智能体（Add to Agent）**。

---

## 第 8 步：检查知识源详情

- 点知识源的 **三个点图标 → 编辑（Edit）**；
- 复核 **名称、描述、所选表** 是否正确；
- 接着按 [`02-知识增强配置.md`](02-知识增强配置.md) 配置同义词、列描述、数据词汇表，显著提升提问理解准确率。

---

## 完成后的验证

用 [`../data/示例问题.md`](../data/示例问题.md) 里的问题实测，例如：

- 「有没有 3 间卧室的房子？」
- 「给我看特拉华州（Delaware）的房源。」
- 「MLS 编号 7 的房子谁是经纪人，电话多少？」

能正确从数据库返回结果即搭建成功。若结果不全或异常，先看 [`03-已知限制与排查.md`](03-已知限制与排查.md)。
